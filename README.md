# 디지털 노마드 지역 추천 대시보드 – 방법론

본 서비스는 디지털 노마드가 **장기 체류하기 좋은 지역**을 찾을 수 있도록  
방문 수요, 생활 인프라, 숙박 기반, 활동 매력도를 종합한 **Nomad Suitability Index (NSI)**를 설계·적용합니다.

## NSI 산출 플로우
입력 데이터 → 핵심 지표(방문자·소비·숙박·활동) → 정규화(0~1) → NSI 산출(가중치+보너스) → 필터/랭킹 → 지도 & 추천

## 1) 입력 데이터 및 전처리

**한국관광데이터랩**  
- 지역별 방문자 수 → 광역 합산  
- (선택) 지역별 관광지출액/검색건수 → 업종/유형별 비중 추정  
- (선택) 인기관광지 현황

**소상공인시장진흥공단**  
- `소상공인시장진흥공단_상가(상권)정보` (공공데이터포털) → 시도 단위 인프라 집계  
  - 카페/편의점/숙박/병원/약국/PC방/세탁/도서관 등 업소 수 및 정규화

**한국철도공사**  
- `한국철도공사_KTX 노선별 역정보` → KTX 역 개수, 접근성 보조 지표

**한국문화정보원**  
- 전국 공유오피스 시설 데이터 → 코워킹 인프라(개수/밀도)

> 전처리: 시도명 정규화(서울·부산… 2글자 키), 필요 컬럼 정리, 결측 처리

---

## 2) 핵심 지표

1. **방문자 점유율** = 지역 방문자 ÷ 전국 방문자  
   → 수요·접근성(사람이 실제로 많이 찾는가)
2. **소비 다양성** = 업종별 비중의 Shannon entropy 정규화  
   → 생활/소비 선택지 폭, 장기 체류 적합성
3. **숙박 지출 비중** = 대분류 ‘숙박업’ 비중  
   → 숙박 인프라 수준
4. **활동 다양성 지수** = 체험/문화/레저의 Shannon entropy 정규화  
   → 머무는 동안 즐길거리 다양성

**선정 이유**  
- 방문자: 기본 수요/접근성  
- 소비 다양성: 생활 편의/선택지  
- 숙박: 체류 인프라  
- 활동 다양성: 여가 매력도

---

## 3) 정규화와 NSI

- 각 지표 **min–max 정규화(0~1)**  
- 가중치: 방문자 **0.3**, 소비 다양성 **0.3**, 숙박 **0.2**, 활동 **0.2**  
- `NSI_base = 0.3*방문자 + 0.3*소비 + 0.2*숙박 + 0.2*활동`  
- `NSI = NSI_base + bonus` (카테고리/인프라 보너스 반영)

---

## 4) 필터 & 보너스 로직

- 분위수 기준: **상위 30% (70분위)** 컷을 기본 기준으로 사용  
- 후보 추출: **ANY**(하나만 충족해도 포함) 기본  
- 보너스:  
  - **카테고리 보너스**(예: 인기/상위랭킹/활동/숙박/소비/교통/코워킹/저렴한비용/빠른인터넷)  
  - **인프라 보너스**(카페/편의점/숙박/병원/약국/PC방/세탁/도서관 정규화 스코어 가산)

---

## 5) 카테고리 정의

### 핵심 지표 기반
- 🔥 현재 인기 지역: 방문자 점유율 상위  
- 🏅 상위 랭킹 지역: NSI_base 상위  
- 🎯 활동 다양 지역: 활동 다양성 상위  
- 🛏️ 숙박 다양 지역: 숙박 비중 상위  
- 🛍️ 소비 다양 지역: 소비 다양성 상위  

### 확장 지표 기반
- 💰 저렴한 비용: `cost_index` 낮을수록 가산  
- 🚀 빠른 인터넷: `internet_mbps` 높을수록 가산  
- (선택) 💨 깨끗한 공기: `air_quality_pm25` 낮을수록 가산  
- (선택) 🛡️ 안전한 지역: `safety_index` 높을수록 가산  
- 💼 코워킹 인프라 풍부: 공유오피스 개수/밀도 정규화 상위

---

## 6) 인프라 지표 산출(소상공인)

- `infra__cafe_count_per10k` = 카페 수 / 업소 10k(정규화 포함)  
- `infra__convenience_count_per10k` = 편의점 수 / 업소 10k  
- `infra__accommodation_count_per10k` = 숙박업소 수 / 업소 10k  
- 병원/약국/PC방/세탁/도서관 등 동일 방식

---

## 7) 키워드/카테고리 탐색

선택 지역(지도 클릭) 기준  
- 업종/유형별 `중분류_전체비중 = (대분류비중×중분류비중)` 합산  
- 상위 중분류를 표 & 바차트로 제공 → **그 지역에서 무엇이 많이 소비/탐색되는지** 파악

---

## 8) 데이터 한계 & 처리

- 일부 광역(예: 서울/경기/강원): 소비/활동 세부 데이터 부족 → 다양성 지표 **NaN/None**  
- 처리: 해당 지표 제외 후 나머지로 NSI 산출  
- 보완 계획: 관광공사/민간 카드매출 연계로 세분화 지표 확보
